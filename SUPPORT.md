# Support

## Usage questions

Use GitHub Discussions if enabled. Otherwise open a clearly titled issue after searching existing documentation and issues.

Include the application version, deployment method, relevant logs with secrets removed, and the troubleshooting steps already attempted.

## Bugs

Use the bug report form. A reproducible example is more useful than a screenshot alone.

## Feature ideas

Use the feature request form and explain the user problem before proposing an implementation.

## Security

Do not use public support channels for vulnerabilities. Follow [SECURITY.md](SECURITY.md).

## Service expectations

This is a community-maintained open-source project. Response times are not guaranteed, and maintainers may prioritize confirmed regressions, security issues, and well-scoped contributions.
